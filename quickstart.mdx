---
title: "Quick start"
description: "Add Datadog Synthetic tests to your GitHub Actions workflow in minutes."
---

This guide walks you through adding the Datadog Synthetics CI GitHub Action to a workflow. By the end, your GitHub Actions job will trigger Synthetic tests and report results back to Datadog on every run.

## Prerequisites

- A Datadog account with at least one [Synthetic test](https://docs.datadoghq.com/synthetics/) already created
- A Datadog [API key and application key](https://docs.datadoghq.com/account_management/api-app-keys/)
- A GitHub repository with Actions enabled

## Setup

<Steps>
  <Step title="Store your keys as GitHub secrets">
    Your Datadog API and application keys must be available to the workflow at runtime. Store them as [encrypted secrets](https://docs.github.com/en/actions/reference/encrypted-secrets) so they are never exposed in your workflow file.

    1. In your GitHub repository, go to **Settings → Secrets and variables → Actions**.
    2. Click **New repository secret** and add a secret named `DD_API_KEY` with your Datadog API key as the value.
    3. Repeat the step to add a second secret named `DD_APP_KEY` with your Datadog application key.
  </Step>

  <Step title="Add the action to your workflow">
    Create or open a workflow file in `.github/workflows/` and add a step that uses `DataDog/synthetics-ci-github-action@v4.0.0`. Pass your secrets as the `api-key` and `app-key` inputs, and list the public IDs of the tests you want to run under `public-ids`.

    ```yaml
    name: Run Synthetic tests using the test public IDs
    jobs:
      e2e_testing:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4
          - name: Run Datadog Synthetic tests
            uses: DataDog/synthetics-ci-github-action@v4.0.0
            with:
              api-key: ${{secrets.DD_API_KEY}}
              app-key: ${{secrets.DD_APP_KEY}}
              public-ids: |
                abc-d3f-ghi
                jkl-mn0-pqr
    ```

    Replace `abc-d3f-ghi` and `jkl-mn0-pqr` with the public IDs of your own Synthetic tests. You can find the public ID for a test on its details page in Datadog, or in the URL when viewing the test.
  </Step>

  <Step title="Commit and push">
    Commit the workflow file and push it to GitHub. GitHub Actions picks up the new workflow automatically.

    Once the job runs, you can follow the test results in two places:

    - **The GitHub Actions job log** — the action streams pass/fail output for each test as results come in.
    - **Datadog** — a CI batch is created in your Datadog organization and linked directly in the job log via the `batch-url` output.

    If any test fails, the action sets the job status to failed, blocking the workflow from succeeding.
  </Step>
</Steps>

<Note>
  If you omit `public-ids`, the action automatically discovers test configuration files matching the glob pattern `{,!(node_modules)/**/}*.synthetics.json` in your repository and runs all the tests defined in those files. This is useful when you manage test configuration as code alongside your application.
</Note>
