---
title: "Global configuration file"
description: "Use a datadog-ci.json file to set defaults for the Datadog Synthetics CI GitHub Action."
---

The `config-path` input points to a JSON configuration file (default: `datadog-ci.json`) that sets default values for `datadog-ci`. This lets you version-control your Synthetics configuration alongside your code instead of embedding all options in your workflow file.

## Configuration file format

Create a `datadog-ci.json` file in your repository with any of the supported fields:

```json datadog-ci.json
{
  "datadogSite": "datadoghq.com",
  "subdomain": "app",
  "batchTimeout": 1800000,
  "failOnTimeout": true,
  "failOnMissingTests": false,
  "failOnCriticalErrors": false,
  "files": [
    "{,!(node_modules)/**/}*.synthetics.json"
  ]
}
```

## Precedence

When the same option is set in multiple places, the following order applies (highest to lowest):

1. **Action inputs** — values set directly in your workflow `with:` block
2. **Config file** — values from `datadog-ci.json` (or the file at `config-path`)
3. **Built-in defaults** — defaults defined by `datadog-ci`

Action inputs always win. The config file fills in values not supplied by the workflow, and built-in defaults apply to anything not set in either place.

## Default path behavior

If no file exists at the default `datadog-ci.json` path and you have not explicitly set `config-path`, the action is silent and continues with built-in defaults.

If you explicitly set `config-path` to a custom location and the file is not found, the action fails with an error:

```
Unable to parse config file! Please verify config path: <your-path>
```

## Custom config path example

```yaml
- name: Run Datadog Synthetic tests
  uses: DataDog/synthetics-ci-github-action@v4.0.0
  with:
    api-key: ${{ secrets.DD_API_KEY }}
    app-key: ${{ secrets.DD_APP_KEY }}
    config-path: './config/datadog-ci.json'
```

<Tip>
  Commit `datadog-ci.json` to your repository root so it is picked up automatically without setting `config-path` in every workflow. This keeps your workflow files concise and lets you update Synthetics configuration in a single place.
</Tip>
